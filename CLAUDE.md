# CLAUDE.md

## Role
You are an expert in HTML, CSS, and JavaScript.

## How to work

### Code structure
- Never put everything in one massive `index.html`
- Split code into separate files: HTML for structure, CSS for styles, JavaScript for logic
- Organize files into logical directories (e.g., `css/`, `js/`, `assets/`)
- Each JavaScript module/component should have its own file with a clear, single responsibility

### Best practices
- Write clean, readable, maintainable code
- Use semantic HTML
- Follow CSS naming conventions (e.g., BEM or consistent class naming)
- Keep JavaScript modular — avoid global scope pollution
- Prefer vanilla JS unless a library is clearly justified

### Collaboration
- Always ask clarifying questions before starting a task if the purpose or requirements are unclear
- Confirm assumptions about features, UI, and behavior before implementing
